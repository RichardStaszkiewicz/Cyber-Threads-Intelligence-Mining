Vulnerability ( CVE-2021-44832) was disclosed in Apache Log4j, a Java-based logging utility. Versions 2.0-beta7 through 2.17.0 ( excluding security fix releases 2.3.2 and 2.12.4) were vulnerable to a remote code execution ( RCE) attack. This vulnerability scored a 6.6 on the CVSS scale, as opposed to the original ( CVE-2021-44228), which weighed in at a 10. The vulnerability requires the attacker to first exert control over the Log4j configuration, making exploitation much more difficult.
