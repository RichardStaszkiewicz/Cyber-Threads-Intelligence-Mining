Log4Shell, a vulnerability found in the Log4j java library was discovered and disclosed in early December, 2021. This vulnerability in Log4j was a zero-day vulnerability now assigned CVE-2021-44228. This critical vulnerability in Log4j, allows an attacker to execute arbitrary code on systems using Log4j to write out log messages. By exploiting the connection between the vehicles and company servers, threat actors can access company assets. This can be achieved by communicating directly with the telematics data servers, to the back-end infrastructure and internal servers, bypassing many existing security controls in place.
